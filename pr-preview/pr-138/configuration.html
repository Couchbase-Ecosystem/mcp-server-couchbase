Configuration

The Couchbase MCP Server can be configured through environment variables or command line arguments set in your MCP client configuration. This allows you to customize server behavior, authentication, transport settings, and more.

Configuration Topics​

  • Environment Variables & Command Line Arguments - Full reference of all configuration options, authentication examples, and transport settings.

  • Read-Only Mode - Control write access to your cluster. Enabled by default for safety.

  • Streamable HTTP Transport Mode - Run the server in Streamable HTTP transport mode for multi-client access.

  • Disabling Tools - Selectively disable individual tools via configuration.

  • Elicitation/Confirmation for Tool Calls - Require user confirmation before executing specific tools.
